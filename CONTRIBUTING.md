# 贡献指南

感谢您对 PlugLock 项目的关注！我们欢迎任何形式的贡献，包括但不限于：

- 提交 Bug 报告
- 提出功能建议
- 编写或改进文档
- 提交代码修复或新功能

## 如何贡献

### 报告 Bug

如果您发现了 Bug，请在 GitHub 上创建一个 Issue，并提供以下信息：

1. 清晰的问题描述
2. 复现步骤
3. 期望的行为
4. 实际的行为
5. 相关环境信息（操作系统、Java 版本等）

### 提出功能建议

如果您有好的想法或改进建议，请创建一个 Issue 并详细描述：

1. 功能需求背景
2. 解决方案建议
3. 可能的影响范围

### 提交代码

1. Fork 项目到您的 GitHub 账户
2. 克隆您的 Fork 到本地
3. 创建一个新的分支用于开发 (`git checkout -b feature/your-feature-name`)
4. 进行编码并提交更改
5. 确保代码符合项目规范
6. 添加相应的单元测试
7. 运行测试确保所有测试通过
8. 提交 Pull Request

### 代码规范

- 遵循项目现有的代码风格
- 添加适当的注释和文档
- 确保新增代码有相应的单元测试
- 所有测试必须通过

### 提交 Pull Request

在提交 Pull Request 时，请确保：

1. PR 有一个清晰的标题和详细的描述
2. 关联相关的 Issue（如果有）
3. 代码已经过测试并且可以正常运行
4. 包含了必要的文档更新

我们会尽快审核您的贡献，谢谢您的参与！
